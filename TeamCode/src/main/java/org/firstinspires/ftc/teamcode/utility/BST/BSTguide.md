# BSTsolver 弹道求解器使用指南

## 概述

`BSTsolver` 是 FTC DECODE 赛季中用于计算小球发射参数的核心算法类。该类基于实测数据插值，能够考虑机器人移动速度对弹道的影响，实现精准的"跑打"功能。

---

## 1. 工作原理

### 1.1 核心思想

BST (Ballistic Solver with Translation) 算法解决的核心问题：

**已知**：目标相对机器人的位置 `(dx, dy)`、机器人移动速度 `(vx, vy)`  
**求解**：发射仰角 `yaw`、发射速度 `speed`、水平旋转角 `roll`

### 1.2 算法流程

```
┌─────────────────────────────────────────────────────────────┐
│                    BST Solver 求解流程                      │
├─────────────────────────────────────────────────────────────┤
│                                                            │
│  输入: vx, vy (机器人速度), dx, dy (目标相对位置)           │
│                    ↓                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  遍历所有可用的仰角 yaw ∈ availableYaws              │   │
│  │  ┌─────────────────────────────────────────────┐    │   │
│  │  │  对每个yaw执行二分搜索，找到满足:            │    │   │
│  │  │     L(t) = d(t)                            │    │   │
│  │  │  其中:                                     │    │   │
│  │  │     L(t) = ||(dx-vx*t, dy-vy*t)||          │    │   │
│  │  │     d(t) = 该仰角下t时刻小球飞行距离         │    │   │
│  │  └─────────────────────────────────────────────┘    │   │
│  │                    ↓                                 │   │
│  │  计算补偿后的目标位置 & residual误差                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                    ↓                                       │
│  选择 residual 最小的 yaw 作为最优解                        │
│                    ↓                                       │
│  输出: Solution(roll, yaw, speed)                         │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 关键数学模型

对于给定仰角 `yaw`，求解方程 `f(t) = L(t) - d(t) = 0`：

```
L(t) = sqrt((dx - vx*t)^2 + (dy - vy*t)^2)  // 补偿后的目标距离
d(t) = interpolator.getDistanceByTime(yaw, t) // 从标定数据插值得到
```

## 2. 标定流程

### 2.1 标定目标

采集不同仰角(yaw)和发射速度下，小球的飞行时间与飞行距离数据。

### 2.2 标定准备

| 物品 | 说明 |
|------|------|
| 平整场地 | 足够长的直线距离（至少10米） |
| 测距工具 | 卷尺或激光测距仪 |
| 计时器 | 精确到毫秒 |
| 笔记本 | 记录数据 |

### 2.3 标定步骤

**Step 1: 确定仰角范围**

选择一组离散的仰角值（建议5-10个），例如：`{45°, 50°, 55°, 60°, 65°}`

**Step 2: 固定仰角，测试不同速度**

对于每个仰角 `yaw`：
1. 设置炮台仰角为 `yaw`
2. 设置发射速度为 `speed`（编码器单位）
3. 发射小球，记录：
   - **飞行时间** `t`（从发射到落地的毫秒数）
   - **飞行距离** `distance`（从发射点到落地点的水平距离，单位：英寸）
4. 重复3次取平均值
5. 更换速度，重复步骤2-4（建议每个仰角测试5-8个速度）

**Step 3: 填写标定数据表**

| 仰角(yaw) | 速度(speed) | 飞行时间(t) | 飞行距离(distance) |
|-----------|-------------|-------------|--------------------|
| 30        | 500         | 0.85        | 45.2               |
| 30        | 600         | 0.95        | 52.8               |
| 30        | 700         | 1.05        | 60.1               |
| ...       | ...         | ...         | ...                |

**Step 4: 生成 data.csv 文件**

按照以下格式写入 `TeamCode/src/main/java/org/firstinspires/ftc/teamcode/utility/LinearInterpolation/data.csv`：

```csv
yaw,speed,distance,t
30,500,45.2,0.85
30,600,52.8,0.95
30,700,60.1,1.05
35,500,48.5,0.92
35,600,56.2,1.02
35,700,64.0,1.12
...
```

> **注意**：单位必须为：yaw(度)、speed(编码器单位)、distance(英寸)、t(秒)

## 3. data.csv 格式详解

### 3.1 文件格式

```csv
yaw,speed,distance,t
30,500,45.2,0.85
30,600,52.8,0.95
45,500,42.5,0.78
...
```

### 3.2 字段说明

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| `yaw` | int | 度 | 炮台仰角 |
| `speed` | int | 编码器单位 | 发射轮速度 |
| `distance` | double | 英寸 | 小球飞行水平距离 |
| `t` | double | 秒 | 小球飞行时间 |

### 3.3 数据要求

1. **完整性**：每个仰角至少需要2个速度点才能进行插值
2. **单调性**：同一仰角下，速度越大，距离应单调递增
3. **覆盖范围**：数据应覆盖比赛中可能用到的全部仰角和速度范围

## 4. API 使用方法

### 4.1 基本使用

```java
// 创建求解器实例
BSTsolver solver = new BSTsolver();

// 机器人当前速度（英寸/秒）
double vx = RobotPosition.getInstance().getVx();
double vy = RobotPosition.getInstance().getVy();

// 目标相对位置（英寸）
double dx = goalX - robotX;
double dy = goalY - robotY;

// 求解
BSTsolver.Solution solution = solver.predict(vx, vy, dx, dy);

if (solution.success) {
    double roll = solution.roll;      // 水平旋转角（弧度）
    int yaw = solution.yaw;           // 仰角（度）
    int speed = solution.speed;       // 发射速度（编码器单位）
    
    // 使用求解结果控制炮台
    turret.rotate_to(Math.toDegrees(roll), yaw);
    shooter.setTargetVelocity(speed);
} else {
    telemetry.addData("BST Error", solution.message);
}
```

### 4.2 在 Turret 类中的集成

`BSTsolver` 在 `Turret.java` 中的实际调用示例（视觉模式）：

```java
// 视觉模式：从AprilTag计算目标位置
Object[] aimResult = aim(true);
double aimRoll = (double) aimResult[1];
double aimYaw = (double) aimResult[2];

double cotYaw = 1.0 / Math.tan(Math.toRadians(aimYaw));
double targetX = HypParams.TagH * cotYaw * Math.cos(Math.toRadians(aimRoll));
double targetY = HypParams.TagH * cotYaw * Math.sin(Math.toRadians(aimRoll));

// 使用BST求解
BSTsolver.Solution solution = bstSolver.predict(
    RobotPosition.getInstance().getVx(), 
    RobotPosition.getInstance().getVy(),
    targetX, targetY
);
```

### 4.3 纯定位模式求解

```java
// 纯定位模式：从地图获取目标位置
double[] goalPos = HypParams.getGoalPosition(targetTagId);
double robotX = RobotPosition.getInstance().getX();
double robotY = RobotPosition.getInstance().getY();
double dx = goalPos[0] - robotX;
double dy = goalPos[1] - robotY;

BSTsolver.Solution solution = bstSolver.predict(
    RobotPosition.getInstance().getVx(),
    RobotPosition.getInstance().getVy(),
    dx, dy
);
```

## 5. Solution 结果结构

```java
public static class Solution {
    public final double roll;     // 水平旋转角（弧度，-π~π）
    public final int yaw;         // 仰角（度）
    public final int speed;       // 发射速度（编码器单位）
    public final boolean success; // 是否求解成功
    public final String message;  // 错误信息（失败时）
}
```

### 5.1 失败情况

| message | 原因 | 解决方法 |
|---------|------|----------|
| `"No yaw data available"` | `data.csv` 为空或未加载 | 完成标定并确保文件存在 |
| `"No valid solution found for any yaw"` | 所有仰角都无法找到满足条件的解 | 增加标定数据覆盖范围 |

## 6. 注意事项

### 6.1 单位一致性

**所有输入必须使用英寸单位！**

| 输入参数 | 来源 | 单位要求 |
|----------|------|----------|
| `vx`, `vy` | `RobotPosition.getVx/Vy()` | 英寸/秒 |
| `dx`, `dy` | 目标相对位置计算 | 英寸 |

### 6.2 数据插值范围

求解器使用线性插值，超出标定范围的数据会返回 `null`。确保标定数据覆盖：

- **仰角范围**：30° ~ 60°（根据实际炮台能力调整）
- **速度范围**：400 ~ 900（编码器单位）
- **距离范围**：覆盖比赛场地最远目标距离

### 6.3 机器人运动假设

算法假设机器人在小球飞行时间内做**匀速直线运动**。如果机器人做加速或转向运动，求解精度会下降。

### 6.4 温度影响

发射轮摩擦力受温度影响，建议在比赛前重新标定或使用温度补偿。

## 8. 故障排除

### 8.1 求解失败

**问题**：`solution.success == false`

**排查步骤**：
1. 检查 `data.csv` 是否存在且非空
2. 检查文件路径是否正确
3. 检查输入参数单位是否为英寸
4. 确认目标距离在标定数据覆盖范围内

### 8.2 精度不足

**问题**：求解结果不准确

**排查步骤**：
1. 增加标定数据点密度
2. 检查发射器是否磨损
3. 确认仰角编码器零点是否正确
4. 考虑空气阻力对远距离的影响

### 8.3 速度抖动

**问题**：同一位置多次求解速度差异较大

**排查步骤**：
1. 增加每个数据点的测试次数（取平均值）
2. 检查发射轮是否打滑
3. 检查电池电压是否稳定
