# 跑打算法：基于时间补偿的二分求解法

## 1. 问题背景

在 FTC 比赛中，机器人需要在移动中向固定目标（如球门）发射小球。由于机器人自身的运动，球在空中的飞行时间内机器人会移动一段距离，导致瞄准点需要提前补偿。传统的静止标定（如查表得到距离→射速、仰角）无法直接用于移动射击。

## 2. Team 1706 的跑打公式

Team 1706 提出了一个经典的运动补偿公式：

\[
|\mathbf{d} - \mathbf{v} \cdot t| = d(t)
\]

其中：
- \(\mathbf{d}\)：射击瞬间机器人到目标的位置矢量（水平面）
- \(\mathbf{v}\)：机器人的速度矢量（水平面）
- \(t\)：球的滞空时间（待求解）
- \(d(t)\)：对于固定仰角，静止标定得到的“滞空时间 \(t\) → 水平飞行距离”函数（单调递增）

**物理意义**：机器人从当前位置发射球，球在空中飞行 \(t\) 秒。在这段时间内机器人运动了 \(\mathbf{v} t\)，因此球击中目标时，机器人已移动到新位置。方程左边是 **球相对于机器人发射点的水平位移**（即球需要飞过的水平距离），右边是 **静止标定中飞行时间 \(t\) 所对应的水平距离**。等式成立时，球恰好命中目标。

> 注：Team 1706 实际使用了迭代法求解该方程（先假设 \(t_0\)，计算补偿后的距离，再查表得 \(t_1\)，重复直到收敛）。但原始方程形式如上。

## 3. 传统迭代法的局限

Team 1706 的迭代法存在两个理论问题：

1. **循环依赖**：需要先假设一个 \(t_0\)（例如用初始距离查表），然后才能计算补偿后的距离，再更新 \(t\)。这本质上是**不动点迭代**，收敛速度是线性的，且可能不收敛（若函数斜率的绝对值 ≥ 1）。

2. **单次修正存在残余误差**：由于第一次用的 \(t\) 是基于机器人静止时的距离，即使迭代多次也需要额外循环，实时性略受影响。

## 4. 二分求解滞空时间（BST）：直接求解耦合方程

我们提出用**二分法**直接求解关于滞空时间t的方程 \(|\mathbf{d} - \mathbf{v} t| = d(t)\)，将其视为关于 \(t\) 的一维非线性方程。

### 4.1 算法步骤

对每个候选仰角 yaw（即每个固定的 \(d(t)\) 函数）：

1. **确定求解区间** \([t_{\min}, t_{\max}]\)  
   - \(t_{\min} = 0\)  
   - \(t_{\max}\) = 该仰角下球的最大滞空时间（对应最远射程，可用插值表上限）
   - 若机器人速度方向朝向目标（\(\mathbf{d} \cdot \mathbf{v} > 0\)），函数 \(L(t) = |\mathbf{d} - \mathbf{v} t|\) 先减后增，为避免非单调，取 \(t_{\max} = \min(t_{\max},\ \frac{\mathbf{d} \cdot \mathbf{v}}{|\mathbf{v}|^2})\)，保证 \(L(t)\) 在区间内单调递减。

2. **检查可解性**  
   计算 \(F(t) = L(t) - d(t)\)。若 \(F(t_{\min}) \cdot F(t_{\max}) > 0\)，则该 yaw 下无解（超出射程），跳过。

3. **二分迭代**  
   设置精度 \(\epsilon\)（如 0.001 秒）和最大迭代次数（如 50 次）。  
   循环：
   - \(t_{mid} = (t_{\min} + t_{\max}) / 2\)
   - 计算 \(F(t_{mid})\)
   - 若 \(|F(t_{mid})| < \epsilon\)，收敛
   - 若 \(F(t_{mid}) > 0\)，则解在左半区间 \(t_{\max} = t_{mid}\)；否则 \(t_{\min} = t_{mid}\)

4. **得到补偿后的参数**  
   收敛后得到 \(t^*\)，则补偿后的水平距离为 \(d(t^*)\)。查表（或由插值函数）得到该距离对应的射速 \(v_{0}\)（或飞轮转速）。水平旋转角：
   \[
   \phi = \text{atan2}(d_y - v_y t^*,\ d_x - v_x t^*) - \theta_{\text{robot}}
   \]

5. **多仰角择优**  
   重复上述过程对每个候选 yaw 求解，根据评价函数（如射速是否在可控范围、方程残差最小等）选择最优的 (yaw, speed, roll) 组合。

### 4.2 单调性假设

为保证二分法有效，需要以下单调性：
- **\(d(t)\) 单调递增**（物理上成立，可通过数据预处理保证）
- **在选定区间内 \(L(t)\) 单调**（通过限制区间到 \(L(t)\) 的单调段实现，参见步骤 1）
- 此时 \(F(t) = L(t) - d(t)\) 严格单调，二分法保证收敛到唯一解。

## 5. 二分法相对 Team 1706 迭代法的优势

| 对比维度 | Team 1706 迭代法 | 二分求解法 |
|---------|----------------|------------|
| **理论循环依赖** | 存在，需多次迭代 | 无，直接求解自洽方程 |
| **收敛速度** | 线性收敛（需要 5~10 次迭代） | 二阶收敛（每次迭代区间减半，30~50 步可达 1e-6 精度） |
| **对初始值依赖** | 敏感，初值不好可能发散 | 不依赖初始值，只要区间包含解 |
| **精度控制** | 间接（通过迭代次数或残差） | 直接（指定绝对误差 \(\epsilon\)） |
| **多仰角择优** | 需外部循环嵌套 | 天然支持，对每个 yaw 独立求解后择优 |
| **实现复杂度** | 简单（仅需查表） | 中等（需处理区间单调性和端点检查） |
| **数值稳定性** | 当函数斜率接近 1 时收敛慢 | 始终稳定，区间缩小率恒定 1/2 |

### 主要优势总结

1. **理论上无偏**：二分法直接求解耦合方程，不存在因初值假设引起的系统偏差。一旦 \(d(t)\) 标定精确，得到的 \(t^*\) 就是方程的真解。

2. **收敛更快**：对于非线性方程，二分法每步将区间缩小一半，迭代 30 次即可达到 \(10^{-9}\) 的相对精度。而线性迭代（不动点）在函数斜率接近 1 时收敛极慢。

3. **鲁棒性强**：只要标定函数单调且区间端点异号，二分法保证收敛，不受函数形状影响。

4. **易于集成**：可以直接替换原有线性插值或迭代模块，只需提供 \(d(t)\) 的连续查询接口（如线性插值或三次样条）。

## 6. 工程注意事项

- **标定数据质量**：\(d(t)\) 必须是严格单调的。若实测有噪声，建议使用单调回归（isotonic regression）或拟合一个单调函数（如多项式 + 单调约束）。
- **实时性**：对每个 yaw 二分 30~50 次，共 3~5 个 yaw，总计算量 <2ms，远小于 FTC 控制周期（典型 10~50ms），完全可行。
- **超射程处理**：若所有 yaw 均无解，则不应发射，可输出警告或回中机构。
- **加速度补偿**：若机器人加速度较大，可在速度模型中增加加速度项：\(\mathbf{v}(t) = \mathbf{v}_0 + \mathbf{a} t\)，此时 \(L(t)\) 表达式为 \(|\mathbf{d} - (\mathbf{v}_0 t + \frac{1}{2}\mathbf{a} t^2)|\)，仍可用二分法（单调性需重新分析）。

## 7. 结论

二分法直接求解 Team 1706 的跑打公式 \(|\mathbf{d} - \mathbf{v} t| = d(t)\)，消除了循环依赖，收敛更快，精度可控，且易于实现多仰角择优。对于 FTC 移动射击场景，该方法在理论严谨性和工程实用性上都优于传统迭代法，是一种值得采用的先进方案。
